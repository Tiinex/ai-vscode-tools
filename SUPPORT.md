# Support Agent Architect

There are several good ways to help this project move forward.

## 1. Help improve the work

You can help by:

- testing the project
- challenging unclear claims
- reporting rough edges in docs or workflows
- contributing fixes, docs, or validation work

That kind of help is valuable because Agent Architect is trying to be unusually explicit about what it really proves and where it still has gaps.

If you are reading older and newer repo materials together, the intended posture is:

- current active docs explain the project's present claim boundary
- preserved benchmarks and support artifacts show what was exercised
- older parked material carries lineage and provenance, not automatic live authority

That distinction is important because the project is trying to increase confidence through explicit evidence, not through aggressive cleanup that would erase history.

## 2. Help sustain the work

Agent Architect is open, but it still has real operating costs.

The main recurring constraint is not just time.
It is also the cost of development tooling and extra token usage, especially when validation, probing, and iteration exceed the normal subscription budget.

In practice, support helps cover things like:

- Copilot usage
- additional token costs
- continued experimentation
- continued verification and documentation work

If you want to help cover that, you can support the project here:

☕ Ko-Fi: https://ko-fi.com/Tiinusen

## 3. Why support matters here

Agent Architect is trying to do something that is still uncommon in AI tooling:

- move important state into visible artifacts
- reduce dependence on hidden platform context
- enforce explicit process instead of optimistic guesswork
- preserve evidence instead of relying on confidence alone

That kind of work benefits from both contributors and financial support.
